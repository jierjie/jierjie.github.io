git08-远程仓库

概念

远程仓库，主要用来，做多人协作开发的时候，代码库的共享工作！

gitHub 就是一个远程仓库！

远程仓库

1. 创建一个远程仓库

创建远程仓库的时候，文件夹名称一般都会用.git来结尾，表示当前文件夹是一个git库！

远程仓库其实就是一个裸仓库，裸仓库，就是直接将.git文件夹中的所有的内容直接暴露出来，不会再有.git文件夹了!

远程仓库的唯一的作用，就是用来做版本库共享。不能再远程仓库中做开发！

远程仓库创建好之后，一般就不会再有别的操作了！

git init –bare

2. 将本地仓库推送到远程仓库中

git push 远程仓库的地址 本地分支名称:远程分支名称

如果本地分支的名称好远程分支的名称一模一样那就可以只写一个分支名
例如： git push https://github.com/avakpan/frontend9.git master

3. 从远程仓库获取版本库到本地

将远程仓库的内容拷贝一份放到指定的本地文件夹当中，如果不指定本地文件夹名称，git会自动创建一个和远程仓库文件夹名称相同的文件夹

git clone 远程仓库地址 本地文件夹名称

4. 从远程仓库获取更新的内容

git pull 远程仓库地址 分支名称

5. 原则

一般情况下 需要向远程仓库推送新的内容之前，需要先获取一次最新的代码
执行push之前要执行一次pull操作

远程仓库刚创建出来的时候是一个空仓库

这时，可以将任意本地仓库通过push命令推送到远程仓库中，但是一旦远程仓库被push过一次之后，仓库就不再是一个空仓库了，之后能够被push到这个仓库中的内容，都是和仓库相关的，不能是任意一个好不相关的仓库，如果使用一个毫不相关的仓库进行push，会被拒绝！

远程地址管理

展示所有的别名

git remote

注意： 如果当前项目是通过git clone从远程仓库获取下来的，那么会默认的有一个远程地址的别名(origin)，指向的就是当时clone这个项目的时候的远程地址

origin https://github.com/avakpan/frontend9.git

添加远程地址别名

git remote add 别名 远程地址

删除远程地址

git remote remove 别名
