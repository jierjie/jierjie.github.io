使用git进行静态页面托管

github

全球最大的源代码托管平台

github和git的关系

git可以用来获取github上的代码，还可以将自己的代码推送到github上

git可以看做是操作github的一个工具！

github静态页面托管服务

一个账户下面只能有一个项目做静态页面托管！

  1. 注册github账号

  2. 创建一个仓库 网页右上角 有一个加号 点击加号 点击 new repository

  3. Repository name 要设置成 自己的用户名.github.io

  4. 点击create repository

  5. 复制 https的连接

  6. 将本地文件夹使用git进行管理， 进行一次提交

  7. 使用git push命令，将本地文件夹中的内容全数推送到github中新创建的仓库中

git push https://github.com/avakpan/123.git master

  1. 在浏览器地址栏中 直接输入 自己的用户名.github.io 就可以访问到自己的网站了！

git push 将本地的仓库推送到github上

git的基本使用

git

分布式版本控制工具！

版本控制工具能够管理的内容

  1. 版本控制工具不仅能够保存文件的内容状态
  2. 很能够保存文件的 是否被删除 是否有新增文件

git的基本使用

git的配置命令，这个命令只需要在一台电脑上执行一次

git config –global user.name pan
git config –global user.email panming@itcast.cn

  1. git init
    git init命令，是将当前所在的文件夹，用git版本控制工具进行管理！

执行这句命令之后，会在当前文件夹中生成一个.git的隐藏文件夹，该文件夹就是git用来存储每次修改的历史记录的版本仓库！

  1. git status
    用来查看当前文件夹中，这些文件的状态，如果文件没有被git管理，会有 untracked files 提示！
  1. git add
    用来将文件添加到git中，让git进行管理！

git add 文件名 文件名
git add * 将所有的文件全部添加到git中进行管理
git add -A 将所有的文件全部添加到git中进行管理

  1. git commit -m “提示信息”
    将当前的文件状态保存一份到版本库当中，生成一个新的版本，历史记录中会有这个版本的记录

  2. git log
    用来查看所有已经保存的版本的内容

  3. git reset –hard 版本号
    将代码恢复到指定的版本

git的原理介绍

分区

  1. 工作目录

  2. 暂存区 在.git文件夹下面 index

  3. 版本库 在.git文件夹下面

文件的状态

untracked 文件新建之后，并未受过管理就是未追踪状态
modified 文件被修改了
deleted 文件被删除了
staged 已暂存，当使用git add命令将文件添加到暂存区之后，这个文件就处于staged
commited 已提交，当使用git commit 提交之后就是 已提交状体

总结

git对于文件的管理，其实就是调用各个命令，将文件在3个分区之间来回传递的过程！

版本控制的介绍

版本控制工具 VCS

Version Control System

将代码每次更改之后的结果进行保存，保存为指定的版本。

VCS可以实现的效果就是，给每一次修改都保存一次历史记录。

版本控制工具分类

本地式版本控制工具

版本库存放在本地，每个人都有自己的版本库，跟别人毫无关系，在单人开发当中，本地版本控制系统可以用来管理代码，但是如果是多人开发，就没办法使用了

集中式版本控制工具

将版本库集中管理，每次修改的内容都需要在这个集中管理的版本库中生成历史记录！ （SVN）

分布式版本控制工具

vi编辑器的使用

vi编辑器的概念

是bash中自带的编辑器，和windows图形界面shell的记事本属于同一个类型的东西。

vi是一个纯命令行的编辑器，没有图形界面，所有的操作都是通过键盘来进行的！

vi启动命令

在bash中直接输入 vi 就可以启动vi编辑器

启动vi编辑器的时候可以加参数

vi 文件路径(文件名)
功能：编辑指定的文件，如果该文件不存在，那么会在保存的时候，自动创建这个文件

vi编辑器的模式

普通模式

当启动vi编辑器之后，编辑器就默认处在普通模式下了
在普通模式下无法进行编辑操作， 所有的键盘按键，几乎都对应了相应的命令

插入模式

只有在插入模式下才可以像普通编辑器一样编辑内容

命令模式

可以执行一些相关的命令，实现一些功能！

模式之间的转换

  1. 普通模式 —> 插入模式
    a 在当前光标的后面插入
    A 在当前行的最后插入
    i 在当前光标的前面插入
    I 在当前行的最前面插入
    o 在当前光标所在行下面新开一行进入插入模式
    O 在当前光标所在行上面新开一行进入插入模式

  2. 插入模式 —> 普通模式
    esc
    ctrl + [

  3. 普通模式 —> 命令模式
    :

  4. 命令模式 —> 普通模式
    把所有输入的命令全部删光，把冒号也删掉，就自动回到普通模式了

esc 两次

  1. 插入模式和命令模式之间不能直接切换，只能先回到普通模式，在进入相应的想切换的模式、

普通模式下的命令

a 在当前光标的后面插入
A 在当前行的最后插入
i 在当前光标的前面插入
I 在当前行的最前面插入
o 在当前光标所在行下面新开一行进入插入模式
O 在当前光标所在行上面新开一行进入插入模式

dd 删除当前行
ndd n表示要删几行
dw 可以删除一个单词

移动指令：
j 下
k 上
h 左
l 右

w 向后移动一个单词
b 想前移动一个单词
e 跳到单词的结尾

0 移动到当前行开始
$ 移动到当前行结束

yy: 复制当前行
p: 粘贴

命令模式下的命令

:w 保存文件
:q 退出编辑器
:wq 保存并退出

:q! 强制退出
:w! 强制保存

如果意外的在编辑的时候关闭了bash，文件没有保存就将bash关闭了，那么当前文件夹下会有个*.swp文件，这个文件保存的就是当时关闭的时候文件的状态，修改的内容全在里面

当你再次使用vi命令打开vi编辑器进行编辑的时候，会出现警告
下面会有相应的选项，我们可以选择recover 输入r 恢复swp文件中的内容 保存并退出
再次打开的时候，还会有警告，需要选择delete 输入d 删除掉这个swp文件 编辑器就恢复正常了

shell的简介

概念

shell是什么？
shell 壳！

shell就是介于用户和操作系统内核之间的桥梁，让用户使用操作系统的时候更加方便！

分类

图形化shell： windows图形化界面
命令行shell: cmd bash（所有的linux操作系统都会自带这个shell） powershell

bash

linux系统下的自带的一个shell

我们可以在bash下执行所有的linux的命令，来实现平时通过图形化界面实现的操作！！

因为在想要执行linux命令，必须是在linux系统的bash环境下。

bash命令的格式

命令名称 -选项 -选项 参数 参数

ls

全称： list
功能： 展示当前文件夹下所有的文件内容

ls -a -a 是一个选项
功能： 显示当前文件夹下所有的内容 包括隐藏的文件

ls -l -l 是一个选项
功能： 将文件以列表的形式进行展示

如果有多个选项 可以依次将选项列在命令后面
ls -a -l

也可以将选项进行合并
ls -al

ls命令也可以有参数
ls 路径
功能 展示指定路径下的所有的文件信息

pwd

全称： print work directory

cd

全称： change directory 使用频率非常高！！

功能： 改变当前所处的工作目录

/ 当前电脑的根目录 里面包含c d 盘
~ 当前用户的根目录

touch

功能：创建文件

touch 文件名称 在当前的文件夹下创建文件

touch 文件的路径 在指定的文件夹下创建文件

mkdir

功能： 创建文件夹

mkdir 文件夹名称 在当前文件夹下创建新的文件夹

mkdir 文件夹路径 在指定的文件夹下创建新的文件夹

clear

功能： 清空屏幕

rm

全称： remove
功能： 删除指定的文件

rm 文件名称 删除当前目录下指定名称的文件

rm 文件路径 删除指定目录下的指定名称的文件

rm -r -r 选项指的是 递归的意思 递归删除！！

rm -f -f 选项是指 force 强制删除

注意： rm 和 rmdir删除的内容不进回收站！！！

rmdir

全称： remove directiory

rmdir 文件夹名称
rmdir 文件夹路径

rmdir只能够删除空文件夹，如果文件夹中有内容，则无法删除！！

cp

全称 copy

功能： 拷贝文件

cp 源文件(名称或者路径) 目标文件(名称或者路径)

cp -r 源文件夹 目标文件夹 可以用来复制文件夹！！

mv

全称 move
功能： 移动文件 类似于剪切

mv 源文件 目标文件

mv 移动文件夹不需要加任何选项，和文件操作一样！

mv也可以用来执行重命名的操作！

cat

功能：展示文件内容

cat 文件名/文件路径

cat是将所有的文件内容一次性的全部展示出来

less

功能：分页展示文件内容

less 文件名/文件路径

翻页： 使用空格 翻一行 方向键 回车键
退出浏览： q

tab键

智能补全

echo

echo用来打印内容

> 输出重定向

所有命令的输出默认都是在屏幕上
我们可以通过 输出重定向命令 将输结果 输出到指定的位置

>：　替换操作
>>: 追加操作
