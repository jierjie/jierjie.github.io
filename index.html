git09-SSH协议介绍

Secure Shell 安全的shell

http协议的格式

http://域名:端口号

ssh 协议格式

用户名@域名

ssh 一般用作服务器的远程管理，我们可以通过ssh安全的shell管理服务器

在bash中提供了一个ssh命令，这个命令就是可以通过ssh协议连接服务器进行管理的！

ssh命令的说明

ssh 用户名@域名

以指定的用户名使用ssh协议登录域名对应的远程服务器

ssh免密登录的原理

在服务器端，有一个名单，名单中存储所有能够登录当前服务器的客户端的公钥！

由于每次客户端在请求登录的时候，会自动将客户端本身的公钥自动发送给服务器，服务器在拿到公钥之后，和名单中的内容进行一一对照，如果有当前这个公钥，那么就直接允许登录了！

在自己电脑上生成 公钥以及私钥！

ssh-keygen -t rsa

执行这个命令的时候，一路默认，一路敲回车
直到看到冒泡泡为止！

生成的秘钥默认在 用户根目录下面的.ssh文件夹中

c:\users\你自己的用户名.ssh

实现服务器免密登录的步骤（了解）

  1. 如果当前电脑从未生成过秘钥，要执行
    ssh-keygen -t rsa
    生成秘钥

  2. 在用户根目录下面的.ssh文件夹中找到生成密钥，id_rsa.pub就是我们需要的公钥文件，复制里面所有的内容

  3. 登录到服务器（这次登录还是要密码的），在用户根目录下面找到.ssh文件夹

  4. 打开.ssh文件夹中的 authorized_keys文件

  5. 将刚才复制的所有的公钥的内容 粘贴到该文件最后，保存退出

  6. 再次登录服务器，便不再需要密码了！！

实现github免密登录的过程

  1. 如果当前电脑从未生成过秘钥，要执行
    ssh-keygen -t rsa
    生成秘钥

  2. 在用户根目录下面的.ssh文件夹中找到生成密钥，id_rsa.pub就是我们需要的公钥文件，复制里面所有的内容

  3. 打开github页面

  4. 点击自己头像 选择下面的settings

  5. 进入到settings页面之后左边栏中会有一个 ssh and gpg keys ，点击

  6. 进入到该页面之后，页面的右上角会有一个 New SSH Key 的绿色按钮， 点击

  7. title文本框中选择性（可写可不写）的输入内容

  8. 在key文本框中，粘贴刚才复制的公钥的全部内容

  9. 点击下面的绿色的 Add SSH Key 按钮

  10. 获取或者push项目的时候，一定要记得选择ssh连接，才可以实现免密登录！

git08-远程仓库

概念

远程仓库，主要用来，做多人协作开发的时候，代码库的共享工作！

gitHub 就是一个远程仓库！

远程仓库

1. 创建一个远程仓库

创建远程仓库的时候，文件夹名称一般都会用.git来结尾，表示当前文件夹是一个git库！

远程仓库其实就是一个裸仓库，裸仓库，就是直接将.git文件夹中的所有的内容直接暴露出来，不会再有.git文件夹了!

远程仓库的唯一的作用，就是用来做版本库共享。不能再远程仓库中做开发！

远程仓库创建好之后，一般就不会再有别的操作了！

git init –bare

2. 将本地仓库推送到远程仓库中

git push 远程仓库的地址 本地分支名称:远程分支名称

如果本地分支的名称好远程分支的名称一模一样那就可以只写一个分支名
例如： git push https://github.com/avakpan/frontend9.git master

3. 从远程仓库获取版本库到本地

将远程仓库的内容拷贝一份放到指定的本地文件夹当中，如果不指定本地文件夹名称，git会自动创建一个和远程仓库文件夹名称相同的文件夹

git clone 远程仓库地址 本地文件夹名称

4. 从远程仓库获取更新的内容

git pull 远程仓库地址 分支名称

5. 原则

一般情况下 需要向远程仓库推送新的内容之前，需要先获取一次最新的代码
执行push之前要执行一次pull操作

远程仓库刚创建出来的时候是一个空仓库

这时，可以将任意本地仓库通过push命令推送到远程仓库中，但是一旦远程仓库被push过一次之后，仓库就不再是一个空仓库了，之后能够被push到这个仓库中的内容，都是和仓库相关的，不能是任意一个好不相关的仓库，如果使用一个毫不相关的仓库进行push，会被拒绝！

远程地址管理

展示所有的别名

git remote

注意： 如果当前项目是通过git clone从远程仓库获取下来的，那么会默认的有一个远程地址的别名(origin)，指向的就是当时clone这个项目的时候的远程地址

origin https://github.com/avakpan/frontend9.git

添加远程地址别名

git remote add 别名 远程地址

删除远程地址

git remote remove 别名

git07-分支介绍

分支的概念

分支可以理解为版本库的一个备份

在版本库中默认有一个主分支，我们可以通过git命令来创建一个新的分支，这个新的分支上面会会有当前这个分支上所有的内容，也就类似于是做了一个备份出来！

分支的操作

  1. 如何创建分支
    git branch 分支名称

  2. 如何切换分支
    默认情况下，我们处在主分支下面，要在其他分支下做开发，首先要切换到该分支

git checkout 分支名

  1. 如何合并分支
    将指定的分支合并到当前分支上！
    git merge 要合并的分支名称

  2. 删除分支
    git branch -d 分支名称

分支的使用场景

多任务并行开发的时候，分支可以做到多个任务之间不会相互影响！

每当要进行一个新任务的开发的时候，都需要创建一个分支出来，单独做开发，功能完成之后，再将分支合并回去。

注意

主分支（master）是在第一次提交完成之后，才有的！！

每次切换分支之前，要将所有的内容做一次提交！！！否则可能丢失修改！！

任意分支都可开辟新的分支，新创建出来的分支，会拥有当前分支中所有的已有内容！

快捷创建并切换分支的命令

git chekcout -b 分支名称

回滚（回退到某个版本）之后，如果后悔了，不想回退了

通过git log -g 查看所要恢复的版本号（短期内可操作）

git branch 分支名 版本号

合并分支！

git的基本使用

git

分布式版本控制工具！

版本控制工具能够管理的内容

  1. 版本控制工具不仅能够保存文件的内容状态
  2. 很能够保存文件的 是否被删除 是否有新增文件

git的基本使用

git的配置命令，这个命令只需要在一台电脑上执行一次

git config –global user.name pan
git config –global user.email panming@itcast.cn

  1. git init
    git init命令，是将当前所在的文件夹，用git版本控制工具进行管理！

执行这句命令之后，会在当前文件夹中生成一个.git的隐藏文件夹，该文件夹就是git用来存储每次修改的历史记录的版本仓库！

  1. git status
    用来查看当前文件夹中，这些文件的状态，如果文件没有被git管理，会有 untracked files 提示！
  1. git add
    用来将文件添加到git中，让git进行管理！

git add 文件名 文件名
git add * 将所有的文件全部添加到git中进行管理
git add -A 将所有的文件全部添加到git中进行管理

  1. git commit -m “提示信息”
    将当前的文件状态保存一份到版本库当中，生成一个新的版本，历史记录中会有这个版本的记录

  2. git log
    用来查看所有已经保存的版本的内容

  3. git reset –hard 版本号
    将代码恢复到指定的版本

git的原理介绍

分区

  1. 工作目录

  2. 暂存区 在.git文件夹下面 index

  3. 版本库 在.git文件夹下面

文件的状态

untracked 文件新建之后，并未受过管理就是未追踪状态
modified 文件被修改了
deleted 文件被删除了
staged 已暂存，当使用git add命令将文件添加到暂存区之后，这个文件就处于staged
commited 已提交，当使用git commit 提交之后就是 已提交状体

总结

git对于文件的管理，其实就是调用各个命令，将文件在3个分区之间来回传递的过程！

vi编辑器的使用

vi编辑器的概念

是bash中自带的编辑器，和windows图形界面shell的记事本属于同一个类型的东西。

vi是一个纯命令行的编辑器，没有图形界面，所有的操作都是通过键盘来进行的！

vi启动命令

在bash中直接输入 vi 就可以启动vi编辑器

启动vi编辑器的时候可以加参数

vi 文件路径(文件名)
功能：编辑指定的文件，如果该文件不存在，那么会在保存的时候，自动创建这个文件

vi编辑器的模式

普通模式

当启动vi编辑器之后，编辑器就默认处在普通模式下了
在普通模式下无法进行编辑操作， 所有的键盘按键，几乎都对应了相应的命令

插入模式

只有在插入模式下才可以像普通编辑器一样编辑内容

命令模式

可以执行一些相关的命令，实现一些功能！

模式之间的转换

  1. 普通模式 —> 插入模式
    a 在当前光标的后面插入
    A 在当前行的最后插入
    i 在当前光标的前面插入
    I 在当前行的最前面插入
    o 在当前光标所在行下面新开一行进入插入模式
    O 在当前光标所在行上面新开一行进入插入模式

  2. 插入模式 —> 普通模式
    esc
    ctrl + [

  3. 普通模式 —> 命令模式
    :

  4. 命令模式 —> 普通模式
    把所有输入的命令全部删光，把冒号也删掉，就自动回到普通模式了

esc 两次

  1. 插入模式和命令模式之间不能直接切换，只能先回到普通模式，在进入相应的想切换的模式、

普通模式下的命令

a 在当前光标的后面插入
A 在当前行的最后插入
i 在当前光标的前面插入
I 在当前行的最前面插入
o 在当前光标所在行下面新开一行进入插入模式
O 在当前光标所在行上面新开一行进入插入模式

dd 删除当前行
ndd n表示要删几行
dw 可以删除一个单词

移动指令：
j 下
k 上
h 左
l 右

w 向后移动一个单词
b 想前移动一个单词
e 跳到单词的结尾

0 移动到当前行开始
$ 移动到当前行结束

yy: 复制当前行
p: 粘贴

命令模式下的命令

:w 保存文件
:q 退出编辑器
:wq 保存并退出

:q! 强制退出
:w! 强制保存

如果意外的在编辑的时候关闭了bash，文件没有保存就将bash关闭了，那么当前文件夹下会有个*.swp文件，这个文件保存的就是当时关闭的时候文件的状态，修改的内容全在里面

当你再次使用vi命令打开vi编辑器进行编辑的时候，会出现警告
下面会有相应的选项，我们可以选择recover 输入r 恢复swp文件中的内容 保存并退出
再次打开的时候，还会有警告，需要选择delete 输入d 删除掉这个swp文件 编辑器就恢复正常了

shell的简介

概念

shell是什么？
shell 壳！

shell就是介于用户和操作系统内核之间的桥梁，让用户使用操作系统的时候更加方便！

分类

图形化shell： windows图形化界面
命令行shell: cmd bash（所有的linux操作系统都会自带这个shell） powershell

bash

linux系统下的自带的一个shell

我们可以在bash下执行所有的linux的命令，来实现平时通过图形化界面实现的操作！！

因为在想要执行linux命令，必须是在linux系统的bash环境下。

bash命令的格式

命令名称 -选项 -选项 参数 参数

ls

全称： list
功能： 展示当前文件夹下所有的文件内容

ls -a -a 是一个选项
功能： 显示当前文件夹下所有的内容 包括隐藏的文件

ls -l -l 是一个选项
功能： 将文件以列表的形式进行展示

如果有多个选项 可以依次将选项列在命令后面
ls -a -l

也可以将选项进行合并
ls -al

ls命令也可以有参数
ls 路径
功能 展示指定路径下的所有的文件信息

pwd

touch 文件名称 在当前的文件夹下创建文件

mkdir 文件夹名称 在当前文件夹下创建新的文件夹

clear

rm 文件名称 删除当前目录下指定名称的文件
